# Автоматизация тестирования записи на курс Инженер по тестированию

## 1. Перечень автоматизируемых сценариев.

### Сценарии навигации к форме

#### Сценарий 1: Поиск курса через поисковую строку (предусловие тестов заполнения формы)

1. Открыть главную страницу Нетологии
2. Нажать кнопку "Каталог курсов"
3. Ввести в поисковую строку название курса
4. Нажать кнопку "Найти курс"
5. Выбрать курс из списка результатов
6. Нажать кнопку "Записаться"

#### Сценарий 2: Поиск через автодополнение

1. Открыть главную страницу Нетологии
2. Нажать кнопку "Каталог курсов"
3. Ввести в поисковую строку название курса
4. Из выпадающего списка выбрать нужный курс
5. Пролистать страницу курса до формы "Запишитесь или получите консультацию"

#### Сценарий 3: Навигация через категорию профессий

1. Открыть главную страницу Нетологии
2. Нажать кнопку "Освоить профессию"
3. В списке найти курс "Инженер по тестированию"
4. Нажать кнопку "Записаться"

#### Сценарий 4: Поиск через фильтр навыков

1. Открыть главную страницу Нетологии
2. Нажать кнопку "Программирование"
3. В меню поиска слева ввести навык "Тестирование"
4. Выбрать пункт "Тестирование ПО"
5. Нажать кнопку "Показать курсы"
6. Выбрать первый курс из списка
7. Нажать кнопку "Записаться"

### Сценарии заполнения формы

#### Сценарий 1: Заполнение формы ожидаемыми (позитивными) данными
1. В поле "Имя" Ввести "Роман"
2. В поле номер телефона ввести "+79991112233"
3. Нажать кнопку "Записаться"
ОР: Появилось уведомление "Ваша заявка принята, ожидайте звонка."

#### Сценарий 2: Ввод минимального значения
1. В поле "Имя" ввести имя "Ян"
2. В поле номер телефона ввести "+683111" (минимальный номер телефона считается из 7 символов. Если можно выполнить регистрацию только на российский номер - об этом должны сообщить перед тестированием)
3. Нажать кнопку "Записаться"
   ОР: Появилось уведомление "Ваша заявка принята, ожидайте звонка."

#### Сценарий 3: Проверка максимально возможного значения
1. В поле "Имя" ввести имя "Александрина-Эльвира-Анастасия-Александрина-Эльвира-Анастасия-Янина"
2. В поле номер телефона ввести "+17849998877555" (самый длинный существующий номер из 15 символов)
3. Нажать кнопку "Записаться"
   ОР: Появилось уведомление "Ваша заявка принята, ожидайте звонка."

#### Сценарий 4: Не заполнение формы (отправка пустой формы)
1. Оставить поля "Имя" и "Телефон" пустыми
2. Нажать кнопку "Записаться"
   ОР: Под каждым из полей появилась надпись "Обязательное поле"

#### Сценарий 5: Ввод 1 символа
1. В поле "Имя" ввести 1 букву
2. В поле номер телефона ввести цифру
3. Нажать кнопку "Записаться"
   ОР: Под полем "Имя" появилась ошибка "Должно быть не короче 2 символов", под полем "Телефон" - "Неверный формат"

#### Сценарий 6: Ввод специфичных символов
1. В поле "Имя" ввести имя "Ёрик" (так же подойдут варианты "Алексей" "Юлий" "Йорик" и т.д. где содержаться уникальные, только в нашем алфавите буквы)
2. В поле номер телефона ввести "8999+332211"
3. Нажать кнопку "Записаться"
   ОР: Под полем "Телефон" появилась ошибка "Неверный формат, + может быть только в начале номера"

#### Сценарий 7: Ввод специфичных символов других стран
1. В поле "Имя" ввести "矢彦" (так же для проверки подойдет иврит, вязь и прочие форматы записи текста разных языков)
2. В поле номер телефона ввести "אבגדהוזחט" (числа на иврите)
3. Нажать кнопку "Записаться"
   ОР: Под полем "Имя" и "Телефон" появилась ошибка "Неверный формат, имя должно быть записано кириллицей" - для поля "Имя" и "Неверный формат, допустим ввод цифр от 0 до 9 и знака +" - для поля "Телефон"

#### Сценарий 8: Завершение ввода имени символом
1. В поле "Имя" ввести имя "Александрина-" 
2. В поле номер телефона ввести "8999332211"
3. Нажать кнопку "Записаться"
   ОР: Под полем "Имя" появилась ошибка "Неверный формат, имя не может заканчиваться символом"

## 3. Перечень необходимых разрешений, данных и доступов.

* **Тестовые данные пользователя:** валидные имя и телефон для заполнения форм записи
* **Разрешение на тестирование:** согласие владельца сайта на проведение автоматического тестирования
* **Доступ к окружению:** доступ к тестовой среде (если предусмотрена)

## 4. Перечень и описание возможных рисков при автоматизации.

| Риск                              | Описание                                                              | Меры минимизации                                                               |
|-----------------------------------|-----------------------------------------------------------------------|--------------------------------------------------------------------------------|
| **Изменение структуры сайта**     | Дизайн и структура сайта могут отличаться от представленных в ТЗ      | Регулярное обновление селекторов, использование относительных путей            |
| **Доступность курса**             | Курс может быть закрыт, переименован или перенесен в другую категорию | Проверка доступности курса                                                     |
| **Изменение атрибутов элементов** | Классы, id и другие атрибуты могут измениться                         | Регулярное обновление селекторов                                               |
| **Капча/защита от ботов**         | Возможное появление механизмов защиты                                 | Согласование с владельцем сайта, использование тестового окружения (если есть) |
| **Нестабильность сети**           | Проблемы с подключением к интернету                                   | Добавление таймаутов и повторных попыток                                       |

## 6. Интервальная оценка с учётом рисков в часах.

### Общая оценка: 2 рабочих дня (16 часов)

### День 1 (8 часов):

* Создание базовой структуры проекта (1 час)
* Реализация Page Objects (3 часа)
* Написание основных тестовых сценариев (4 часа)

### День 2 (8 часов):

* Отладка и исправление тестов (3 часа)
* Рефакторинг и оптимизация кода (2 часа)
* Доработка под неучтенные сценарии(3 часа)
